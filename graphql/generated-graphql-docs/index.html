Description

Unified Search GraphQL API is meant for searching several non-GraphQL data sources available in City of Helsinki using GraphQL.

The queryable data include e.g. administrative divisions, ontology words/trees and venues.

Source code

https://github.com/City-of-Helsinki/unified-search

Queries

administrativeDivisions

Description

Get Helsinki/Finland administrative divisions i.e. neighborhoods (=kaupunginosat), sub-districts (=osa-alueet), municipalities (=kunnat) etc.

Based on imported data from django-munigeo:

  • "geo_import finland --municipalities"
  • "geo_import helsinki --divisions"

Used django-munigeo importers:

  • geo_import
    • finland
    • helsinki
Response

Returns [AdministrativeDivision]

Arguments
Name Description
helsinkiCommonOnly - Boolean If true, return only Helsinki municipality's neighborhoods and sub-districts (=Helsingin kunnan kaupunginosat ja osa-alueet), otherwise return also all the municipalities of Finland.

Example

Query
query administrativeDivisions($helsinkiCommonOnly: Boolean) {
  administrativeDivisions(helsinkiCommonOnly: $helsinkiCommonOnly) {
    id
    type
    municipality
    name {
      fi
      sv
      en
    }
  }
}
Variables
{"helsinkiCommonOnly": true}
Response
{
  "data": {
    "administrativeDivisions": [
      {
        "id": "ocd-division/country:fi/kunta:helsinki/kaupunginosa:kruununhaka",
        "type": "neighborhood",
        "municipality": "Helsinki",
        "name": {"fi": "Kruununhaka", "sv": "Kronohagen", "en": null}
      }
    ]
  }
}

Queries

unifiedSearch

Response

Returns a SearchResultConnection

Arguments
Name Description
text - String Free form query string, corresponding to user search input
ontology - String Optional, filter to match only these ontology words
administrativeDivisionIds - [ID] Optional, filter to match only these administrative divisions
ontologyTreeIdOrSets - [ID!] Optional, filter to match at least one ontology tree ID from each list
ontologyWordIdOrSets - [ID!] Optional, filter to match at least one ontology word ID from each list
providerTypes - [ProviderType] Optional, filter to match any of these provider types
serviceOwnerTypes - [ServiceOwnerType] Optional, filter to match any of these service owner types
targetGroups - [TargetGroup] Optional, filter to match any of these target groups
mustHaveReservableResource - Boolean Optional, filter to show only venues that have at least one reservable resource. If not given or false, all venues are shown.
index - UnifiedSearchIndex Optional search index.
after - String Optional pagination variable, match results after this cursor.
first - Int Optional pagination variable, limit the amount of results to N.
languages - [UnifiedSearchLanguage!]! Targets the search to fields of specified language. Default = [FINNISH, SWEDISH, ENGLISH]
openAt - String Return only venues that are open at the given moment. In addition to ISO 8601 datetimes, accepts values conforming to Elastic Search date math https://www.elastic.co/guide/en/elasticsearch/reference/7.x/common-options.html#date-math like "now+3h". When there is a datetime provided without a timezone offset, "Europe/Helsinki" will be assumed as the time zone.
orderByDistance - OrderByDistance Order results by distance to given coordinates. Mutually exclusive with other orderBy* parameters.
orderByName - OrderByName Order results by venue name in language given as the first value in "languages" argument. Mutually exclusive with other orderBy* parameters.
orderByAccessibilityProfile - AccessibilityProfile

Optional, order venues by given accessibility profile's shortcomings, first the results that have no accessibility shortcomings for the given profile, then the ones with 1 shortcoming, 2 shortcomings, ..., N shortcomings and last the ones with unknown shortcomings (i.e. no data for or against).

Mutually exclusive with other orderBy* parameters.

unifiedSearchCompletionSuggestions

Response

Returns a SearchSuggestionConnection

Arguments
Name Description
prefix - String Free form query string, corresponding to what the user has typed so far
languages - [UnifiedSearchLanguage!]! Limits the result set into the specified languages. Default = [FINNISH, SWEDISH, ENGLISH]
index - UnifiedSearchIndex Optional search index.
size - Int Optional result size. Default = 5

Example

Query
query unifiedSearchCompletionSuggestions(
  $prefix: String,
  $languages: [UnifiedSearchLanguage!]!,
  $index: UnifiedSearchIndex,
  $size: Int
) {
  unifiedSearchCompletionSuggestions(
    prefix: $prefix,
    languages: $languages,
    index: $index,
    size: $size
  ) {
    suggestions {
      label
    }
  }
}
Variables
{
  "prefix": "xyz789",
  "languages": ["FINNISH", "SWEDISH", "ENGLISH"],
  "index": "administrative_division",
  "size": 5
}
Response

  "data": {
    "unifiedSearchCompletionSuggestions": {
      "suggestions": [Suggestion]
    }
  }
